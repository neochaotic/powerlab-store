## 0.1.0 — 2026-05-20
### Added
- Initial repo bootstrap: license (AGPL-3.0 code + CC-BY-SA-4.0 content),
CONTRIBUTING with app submission flow + trust matrix, SECURITY policy,
Code of Conduct.

Documentation: docs/schema.md (canonical x-powerlab spec),
docs/curation-criteria.md (registry trust + safety rules),
docs/icon-guidelines.md (rehost policy, attribution).

Scripts: validate.py (schema + safety + registry trust + .sh ban +
digest-pin strict), check-safety.sh (compose hardening mirror of core),
pin-digests.sh (BSD/GNU sed-safe), build-index.sh (no tarball — just
index.json), cut-release.sh (mandatory pre-tag step).

CI workflows: validate.yml (schema + safety + changie fragment),
icon-lint.yml (presence + size cap + SVG sanity), smoke-spinup.yml
(Tier 3: PR-touched apps actually `docker compose up -d`, healthcheck,
port probe, log error grep), release.yml (validate + GitHub release
on tag push).

Templates: PR template with split checklists, new-app + app-bug issue
templates.

Changie config + initial CHANGELOG header. VERSION at 0.1.0-dev.

- Phase 1 catalog import: 135 apps converted from the external
curation pipeline artifact (139 source apps, 4 deliberately
excluded for security/schema violations — see PR description).

Composition: ~72 from Umbrel-derived composes, ~60 from
CasaOS-AppStore, 3 PowerLab-authored (Ollama CPU/Nvidia/AMD).
Image policy: all images on the trusted registry whitelist,
zero hooks/exports.sh, zero privileged/sock/host-namespace,
volumes rewritten to /DATA/PowerLabAppData/<id>/.

Apps still need digest-pinning (run scripts/pin-digests.sh)
before v0.1.0 cut. PR shipping as DRAFT until that's complete.

- Per-app description.md quickstarts for all 135 catalog apps.

Each description.md now includes:
- 2-3 sentence description of what the app does and who it's for
- First-steps URL with port (or connection string for headless backends)
- Default login credentials where applicable (with change-immediately note)
- Data path describing what is stored under /DATA/PowerLabAppData/<app>/

Also added:
- scripts/gen-descriptions.py — reproducible generator; re-run after adding apps
- CONTRIBUTING.md description.md template with format rules

- Store↔core bundle-compat contract CI gate:

scripts/check-bundle-compat.py — pre-merge validator that verifies every
Apps/<id>/docker-compose.yml will survive the bundle-store.sh format
conversion:
- title/tagline must be non-empty flat strings (converted to {en_us:…})
- port_map must have ≥1 http/https entry with a host port, OR headless: true
- icon must be a dict with 'file' key pointing to an existing file on disk

Wired into validate.yml as a mandatory CI step on every PR.
Counterpart: scripts/check-bundle-integrity.sh in PowerLab core validates
the bundled community-catalog/ after bundle-store.sh runs.

### Fixed
- Smoke CI + catalog correctness pass after Phase 1 import:

Compose bugs (would break in production):
- linkwarden: add missing top-level networks.linkwarden-net definition
- sure: add missing top-level networks.sure_net definition
- petio: add missing top-level networks.petio-network definition
- yamtrack: correct REDIS_URL from legacy docker-compose-v1 hostname
  (yamtrack_redis_1) to docker-compose-v2 service name (redis)
- mainsail: remove config.json bind-mount that Docker materialises as a
  directory, causing OCI mount failure; default embedded config is sufficient

Smoke workflow improvements:
- Auto-skip hardware-only apps (AMD ROCm /dev/kfd, Raspberry Pi
  /dev/vchiq+/dev/vcsm, NVIDIA GPU runtime) — these require physical
  devices not present on GitHub-hosted runners
- Pre-create /DATA/... bind-mount paths before compose up with sudo;
  postgres data dirs chowned to UID 999 so postgres:16 initdb can chmod
  them (GitHub Actions runner creates bind dirs as root otherwise)
- dcrpulse: relax depends_on condition from service_healthy to
  service_started — dcrd requires blockchain sync before healthcheck
  passes (hours+), dcrwallet/dashboard use restart:on-failure to retry
